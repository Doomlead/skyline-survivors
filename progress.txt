## What We Should Work On Next

Given the current implementation state and the updated design document, the highest-value next task is still to build the **Reclaimer assault recovery loop** first.

### Why this should be next
- It is now a central assault identity requirement in the design document, not optional polish.
- It unblocks several dependent requirements (radar signaling, scoring hooks, operative priority behaviors, and assault flow clarity).
- It resolves a major gameplay gap where assault recovery currently does not match the specified "destroy the Reclaimer" rule.
- It creates the backbone needed for assault-only UI language (countdown pressure, radar target identity, recovery success/failure states).

### One-Task Update (Recommended Next Task)

**Task:** Implement **Reclaimer Recovery v1** for assault missions.

**Definition of Done:**
1. On Aegis destruction in assault mode, spawn Reclaimer with mission announcement/telegraph and radar marker.
2. Add extraction behavior + 30s countdown pressure and failure handling (escape -> delayed respawn cycle).
3. Destroying Reclaimer reconstructs/returns Aegis as the only assault recovery path.
4. Disable Reclaimer spawning during assault interior breach phase (explicit edge-case guard).
5. Add automated tests covering spawn gating, recovery success path, extraction failure/respawn, and interior no-spawn behavior.

### Recommended Follow-Up After That
1. Refactor Stasis Array destruction output to captive release + required hangar conversion (remove direct-operative bypass).
2. Hook full Reclaimer and liberation parity state into ammo/scoring/HUD/radar outputs.
3. Productize Last Stand + Assault Only challenge surfaces with per-mode records.
4. Add weather/time-of-day reactivity pass.

## Suggested Next Workstream (My Recommendation)

If we want maximum design-document compliance per unit of effort, we should execute the next work as a single **Assault Identity Compliance Sprint**:

1. **Reclaimer v1 (must-have)** — complete assault recovery identity.
2. **Stasis parity fix (must-have)** — make liberation economy consistent across sources.
3. **HUD/Radar objective clarity (must-have)** — make both mechanics readable under pressure.
4. **Challenge mode productization (high value)** — expose replay loop once assault identity is stable.

This sequence minimizes rework because HUD/objective text can be implemented once the Reclaimer + liberation parity data model is finalized.
